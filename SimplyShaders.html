Introduction

I have always been interested in Minecraft mod development but never really knew what to make. I was introduced to Cosmic Reach and started learning how to make java mods thanks to the community. After a few test mods, I decided to try and create Cosmic Reach's version of Iris because it was a way for me to learn OpenGL and get a cool result that other people could enjoy too. It took a few attempts to get everything working correctly but I managed to make a working mod that i am happy with.

The mod works around shaderpacks - which are a zip file or folder that is placed in the correct directory. The shaderpack then contains whichever shaders in the base game you want to replace, along with aditional shaders that are specific to the mod, such as post processing shaders. It also contains configuration info like setting render stage resolution, enabling mip mapping, shadow rendering pass, or the details for configuring the shaderpack settings

Shaderpack that comes with the mod https://github.com/Shfloop/SimpleShader

System Architecture

[Caption: architecture / pipeline diagram]

Components

  • Configurable Render Pipeline — Changes what the pipeline should be doing based on the loaded shaderpack
  • Shaderpack Parsing — Loads a shaderpack folder or zip file and formats it to be able to configure the render pipeline
  • User Interface — Provides a customizable UI for individual shaderpacks which allow packs to define custom settings to be applied during render pipeline configuration

Data Flow

  1. User selects which shader pack they want to use or edit the settings of the pack they are using
  2. The rendder pipeline is changed to run the pack

Lessons & Next Steps

What went well

  • I learned a lot about engine development, graphics, and java modding
  • I made a mod that works and finished what I set out to do.

What I’d improve

  • Their are many features missing that would make creating shaderpacks easier and more enjoyable so if i continue that is something that would be needed.
  • I would like to make it a more abstracted pipeline because currently its a modification of the base games pipeline resulting in massive changes when the games pipeline changes. Making it more abstract and reimplimenting the games pipeline would make it a lot easier to update along side the game.

Next steps
